## 2. 시나리오 (5~10줄)
* 이 프로그램은 매달 식비 부족에 시달리는 자취생들을 위해 냉장고 속 식재료와 지출을 스마트하게 관리해 주는 '자취 라이프 매니저' 시스템입니다.
* 가장 먼저, 이번 달의 소중한 식비 예산을 설정하고 오늘 새로 산 식재료의 이름과 가격, 그리고 남은 유효기간(혹은 무게)을 기록하는 것으로 관리를 시작합니다. 그다음에는 입력한 정보를 바탕으로 이 재료가 내 전체 예산에서 어느 정도의 비중을 차지하는지 '식비 점유율'을 계산해 봅니다. 이렇게 계산된 점유율이 너무 높으면 '지출 주의' 경고를, 적절하면 '합리적 소비'라는 판정을 내려줍니다. 만약 가격도 저렴한데 영양가까지 높은 '갓성비' 재료를 샀다면 '알뜰 자취왕'이라는 특별한 칭호를 부여하여 동기부여를 해줄 것입니다. 나중에는 단순히 하나씩 기록하는 것이 아니라, 냉장고 속의 수많은 재료를 커다란 바구니(리스트)에 담아 유통기한 순으로 정렬하거나 전체 남은 예산을 실시간으로 조회할 수 있게 만들 것입니다. 최종적으로는 사용자가 메뉴판에서 번호를 눌러 기능을 선택하고, '자취 종료(로그아웃)'를 하기 전까지는 계속해서 식재료를 관리할 수 있는 완벽한 자취 보조 소프트웨어로 완성될 계획입니다.

## 🟩 [2차 과제: V1.0] 입출력 + 리스트 + 조건문 - 향후 작성 예정
### **✨2차 과제 내용:**
* 이번 과제는 **[자취 라이프 매니저] 냉장고 관리 시스템 V1.0** 구현으로, 사용자로부터 식비 예산과 식재료 정보를 입력받아 리스트에 저장하고, 지출 금액과 예산 점유율을 계산하는 프로그램을 만드는 것입니다.
* `input()`으로 사용자 이름, 이번 달 식비 예산, 식재료 이름, 가격, 남은 유통기한, 영양 점수를 입력받아 리스트에 저장했습니다.
* `for`문을 사용하여 최소 3개의 식재료 데이터를 입력받고, `append()`를 활용해 식재료 이름과 가격, 유통기한, 영양 점수를 각각 리스트에 추가했습니다.
* `sum()`, `len()`, `max()`, `sort()`, `insert()` 등을 활용하여 리스트 데이터를 조작하고, 총 지출 금액과 남은 예산, 식비 점유율을 계산했습니다.
* `if-elif-else` 조건문을 사용하여 지출 등급을 **알뜰 소비 / 적정 소비 / 과소비 주의**로 판정했습니다.
* `and`, `or`, `not` 논리 연산자와 중첩 조건문을 활용하여 예산과 영양 점수를 함께 고려한 **알뜰 자취왕** 특별 칭호를 부여했습니다.

    
### **🤖 AI 파트너십 과정**
1. **문제 1: 변수 이름 오타로 인해 없는 항목이라고 뜨는 문제**
    * **원인:** 리스트나 변수 이름을 선언할 때 사용한 이름과 출력 또는 계산할 때 사용한 이름이 서로 달라서 오류가 발생했습니다. 예를 들어 `food_prices`로 만든 리스트를 다른 줄에서 `food_price`처럼 잘못 입력하면 파이썬이 해당 변수를 찾지 못했습니다.
    * **해결:** 변수 이름을 한 줄씩 확인하면서 선언한 이름과 사용하는 이름을 모두 동일하게 수정했습니다. 이번 경험을 통해 변수 이름은 처음부터 의미 있게 정하고, 같은 이름을 끝까지 일관되게 사용하는 것이 중요하다는 것을 알게 되었습니다.

2. **문제 2: 필요한 코드 한 줄을 빠뜨려 실행 결과가 원하는 대로 나오지 않은 문제**
    * **원인:** 식재료 정보를 입력받은 뒤 리스트에 추가하는 `append()` 줄이나, 총 지출 금액을 누적하는 `total_price += price` 줄을 빠뜨려 계산 결과가 제대로 반영되지 않았습니다.
    * **해결:** 입력 → 리스트 저장 → 계산 → 출력 순서로 코드 흐름을 다시 확인하며 빠진 줄을 추가했습니다. 특히 반복문 안에서 입력받은 값이 리스트에 저장되고 있는지, 총 지출 금액에 누적되고 있는지 확인했습니다.

3. **문제 3: input으로 입력받은 숫자가 계산에 바로 사용되지 않는 문제**
    * **원인:** `input()`으로 입력받은 값은 기본적으로 문자열이기 때문에 가격이나 예산 계산에 바로 사용할 수 없었습니다.
    * **해결:** 예산과 가격은 `int()`, 영양 점수는 `float()`로 형변환하여 계산이 가능하도록 수정했습니다.

4. **문제 4: 조건문 결과가 예상과 다르게 출력되는 문제**
    * **원인:** `if`, `elif`, `else`의 순서를 잘못 배치하거나 조건 범위를 애매하게 작성하여 원하는 등급이 출력되지 않았습니다.
    * **해결:** 식비 점유율이 큰 경우부터 차례대로 검사하도록 조건문의 순서를 정리했습니다. 또한 `and`, `or`, `not` 연산자를 사용할 때 각각의 조건이 어떤 의미인지 확인하며 수정했습니다.
